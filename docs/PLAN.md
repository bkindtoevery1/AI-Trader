# 운영 플랜

## 1. API 연동

- OAuth2 Client Credentials로 토큰을 발급한다.
- 시세와 캔들은 토큰만으로 조회한다.
- 계좌, 보유 자산, 매수 가능 금액, 주문은 `X-Tossinvest-Account` 헤더를 추가한다.
- 429 응답은 `Retry-After` 기준으로 재시도한다.

## 2. 전략

- 초기 전략은 단순하고 검증 가능한 룰로 시작한다.
- 매수: 단기 SMA가 장기 SMA를 상향 돌파하고 RSI가 과열 구간이 아닐 때.
- 매도: 단기 SMA가 장기 SMA를 하향 돌파하거나 RSI가 과열 구간일 때.
- 전략 변경은 백테스트 수익률과 최대 낙폭을 같이 보고 결정한다.

## 3. 일일 개선 루프

1. 장 종료 후 캔들 데이터를 업데이트한다.
2. 현재 전략으로 백테스트한다.
3. 후보 파라미터를 그리드 탐색한다.
4. 기준 전략 대비 개선된 후보만 리포트한다.
5. 사람 검토 후 `config/strategy.yaml`에 반영한다.

## 4. 리스크 관리

- 기본 실행은 주문하지 않는다.
- 실거래는 설정과 CLI 플래그가 둘 다 켜져야 한다.
- 주문 금액, 일일 주문 수, 현금 보존 비율을 강제한다.
- 고액 주문과 레버리지/파생 전략은 초기 범위에서 제외한다.

## 5. 배포

- CI는 Python 테스트와 웹 빌드를 항상 실행한다.
- GitHub Pages 배포 워크플로는 리포트 JSON을 생성한 뒤 대시보드를 정적 파일로 배포한다.
- 일일 리포트 워크플로는 스케줄 실행과 수동 실행을 모두 지원한다.
